How ZAGA Observability Solution Works

Our Observability solution operates using OpenTelemetry, an open-source observability framework. It offers a suite of APIs, libraries, agents, instrumentation, and standards to facilitate observability across various distributed system services and components.

Instrumentation:

Manual Instrumentation:

Developers leverage OpenTelemetry libraries to instrument their code, collecting trace information, metrics, and logs within the application's codebase.

Auto Instrumentation:

Post-deployment, the application can be effortlessly instrumented.

Tracing:

OpenTelemetry enables users to trace the flow of requests as they move through diverse services and components. Each trace comprises spans, representing individual units of work or operations within a service and containing information such as start and end times, duration, associated attributes, and context propagation details.

Metrics:

OpenTelemetry supports the collection of metrics, allowing monitoring and measurement of various aspects of application performance, resource utilization, and custom business metrics. Metrics offer aggregated and time-series data to identify trends and anomalies.

Logging:

OpenTelemetry streamlines the generation of structured logs that offer additional context during debugging and troubleshooting. These logs capture pertinent information about events, errors, or significant activities within the application.

Exporter:

OpenTelemetry supports various exporters to send the collected telemetry data to observability backends, such as tracing and metrics databases. The Observability solution exports the data to Kafka.

Kafka:

Kafka serves as the backbone for efficient collection and distribution of telemetry data, enabling seamless data processing and streamlined communication within the Observability infrastructure.

Observability Backend:

Our observability backend receives and stores the telemetry data for analysis and visualization. This data is then queried by another Backend application, and the data is viewed using the front-end application.
